// 04 · Hybrid AI System

Hybrid LLM + ML Caffeine Recommender

A proof-of-concept recommendation system for data-scarce domains. It starts with no real user data, uses an LLM to convert natural language into validated JSON, prototypes a global model from synthetic rule-based data, then evolves through user feedback into personalised recommendations.

→ View on GitHub → Read on Medium
Core Idea
LLM acts as a deterministic natural-language-to-JSON parser
Random Forest validates the global recommendation pipeline using synthetic data
Per-user linear models re-rank drinks from real feedback
Aggregated updates improve the global model over time
// Personal Model · Live Feedback Re-rank

Zero-Data Start

Drink profiles and rule-based synthetic labels test the recommendation pipeline before any real feedback exists.

LLM JSON Parser

Strict prompt engineering, fixed schema constraints and low temperature convert messy user intent into reliable structured input.

Explainable Output

Top recommendations are fed back into the LLM so it can explain the ranked choices conversationally without overriding them.

Personal Feedback Loop

1-5 user ratings with epsilon-greedy exploration to reduce exposure bias, training per-user models once enough data exists.

Federated-Style Updates

Aggregated user model coefficients with sample-aware weighting, inspired by FedAvg/FedNova, to improve the global starting point.

// 05 · Research Interest

Responsible AI & Safety

I am especially interested in AI systems that are useful without becoming opaque or careless. Across my projects, that means treating reliability, explainability, validation and user safety as design requirements rather than afterthoughts.

Principles I Build Around
Constrain model outputs when structure matters
Keep humans aware of uncertainty and limitations
Use feedback loops carefully to reduce bias and drift
Document assumptions, data choices and validation clearly

Structured LLM Outputs

Natural language is converted into a fixed JSON schema with validation so downstream ML receives predictable, auditable inputs.

Transparent Ranking

Recommendations are ranked through explicit features, rules, feedback and model scores - explained conversationally without LLM override.

Bias-Aware Feedback

Epsilon-greedy exploration reduces exposure bias and personal models are weighted carefully before influencing global behaviour.

Data Integrity

The collisions pipeline documents navigation, preprocessing, validation splits, missingness, VIF pruning and a no-LLM declaration.

Controlled Generation

LLM commentary is decoupled from driving, uses cooldowns, and applies soft stopping via logit biasing to keep generation bounded.
