// 04 · Hybrid AI System

Hybrid LLM + ML Recommendation System

A hybrid recommendation system for data-scarce domains, combining LLM-based input parsing, synthetic-data prototyping and feedback-driven personalisation. The system starts without real user data, validates a global model using rule-based synthetic labels, then adapts through per-user feedback into personalised recommendations.

// Personal Model · Live Feedback Re-rank

Zero-Data Start

Drink profiles and synthetic rule-based labels validate the recommendation pipeline before real user feedback exists.

LLM-to-JSON Parsing

Strict prompting, schema validation and low-temperature generation convert messy user intent into reliable structured input.

Explainable Recommendations

Ranked recommendations are passed back to the LLM for conversational explanations, without letting it override the model outputs.

Personal Feedback Loop

User ratings and epsilon-greedy exploration reduce exposure bias while training personalised ranking models over time.

Federated-Style Updates

Sample-weighted user model updates, inspired by FedAvg and FedNova, help improve the global model without relying on one user's data.

// 05 · Research Interest

Responsible AI & Safety

I'm interested in AI systems that remain reliable, explainable and controllable as they become more capable. Across my projects, that means treating validation, transparency and user safety as core engineering requirements rather than afterthoughts.

Principles I Build Around
Constrain model outputs where reliability matters
Make uncertainty and system limitations visible
Design feedback loops to reduce drift and bias
Document assumptions, validation and data choices clearly

Structured LLM Outputs

Natural language is converted into validated JSON schemas so downstream systems receive predictable and auditable inputs.

Transparent Ranking

Recommendations are ranked through explicit features, feedback and model scores, then explained conversationally without allowing the LLM to override the ranking.

Bias-Aware Feedback

Epsilon-greedy exploration reduces exposure bias, while weighted aggregation prevents individual users from disproportionately shaping the global model.

Data Integrity

The collisions pipeline emphasises reproducibility through documented preprocessing, validation splits, feature pruning and transparent ML workflows.

Controlled Generation

LLM commentary is separated from driving logic and constrained using cooldowns and logit biasing to keep generation controlled and bounded.
